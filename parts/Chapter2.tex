\section{Автоматическая векторизация программного кода}
\label{sec:Chapter2} \index{Chapter2}

\subsection{SIMD как аппаратная основа векторизации}

SIMD (Single Instruction Multiple Data)~\cite{flynn1972some} представляет собой модель вычислений, 
при которой одна инструкция выполняет одинаковую операцию над несколькими элементами данных, 
размещёнными в векторных регистрах. 
Данный подход позволяет повысить производительность за счёт параллельной обработки независимых данных.

Векторный регистр состоит из нескольких одинаковых по размеру элементов, называемых линиями. 
Каждая линия содержит одно значение обрабатываемого типа данных. 
Стандартная SIMD-инструкция выполняет одну и ту же операцию одновременно над всеми линиями регистра. 
Число линий определяется шириной регистра и размером элемента данных. 
Например, 128-битный регистр может содержать четыре 32-битных целых числа или два 64-битных числа.

Простейший пример SIMD-вычислений — поэлементное сложение двух массивов:
\[
c_i = a_i + b_i,\quad i = 0,\dots,n-1
\]

При использовании SIMD вычисления выполняются над вектором из \(m\) элементов одновременно:
\[
\mathbf{c}_i = \mathbf{a}_i + \mathbf{b}_i,\quad \mathbf{a}_i, \mathbf{b}_i, \mathbf{c}_i \in \mathbb{R}^m
\]

Современные процессорные архитектуры поддерживают SIMD-вычисления через специализированные наборы инструкций:
\begin{itemize}
    \item ARM: NEON и SVE;
    \item x86: SSE, AVX, AVX2, AVX-512.
\end{itemize}

По функциональному назначению SIMD-инструкции можно условно разделить на следующие классы:
\begin{itemize}
    \item арифметические операции (сложение, умножение и т.д.);
    \item логические операции;
    \item операции перестановки и упаковки данных (shuffle, permute);
    \item операции загрузки и сохранения векторных данных;
    \item редукционные операции.
\end{itemize}

Эффективное применение SIMD требует выполнения ряда условий:
\begin{itemize}
    \item отсутствие зависимостей по данным между обрабатываемыми элементами;
    \item согласованность типов и размеров операндов;
    \item возможность группировки скалярных операций в векторные пакеты.
\end{itemize}

Несмотря на наличие аппаратной поддержки, использование SIMD-инструкций на уровне прикладного программирования связано с рядом трудностей, включая зависимость от конкретной архитектуры и необходимость явного управления векторными регистрами.

Поэтому важную роль играют методы автоматической векторизации, позволяющие компилятору выявлять участки кода, пригодные для преобразования в SIMD-форму.

\subsection{Подходы к автоматической векторизации}

Автоматическая векторизация представляет собой класс оптимизаций компилятора, направленных на преобразование скалярного программного кода в эквивалентную форму, использующую SIMD-инструкции.

В современных компиляторах можно выделить два основных подхода к автоматической векторизации:
\begin{itemize}
    \item цикловая векторизация;
    \item SLP-векторизация.
\end{itemize}

\subsection{Цикловая векторизация}

Цикловая векторизация ориентирована на выявление параллелизма между итерациями циклов. Рассмотрим пример:

\begin{center}
\begin{tabular}{c}
\begin{lstlisting}[language=Go, captionpos=b]{senter}
for i := 0; i < n; i++ {
    c[i] = a[i] + b[i]
}
\end{lstlisting}
\end{tabular}
\end{center}

Данный цикл не содержит межитерационных зависимостей, что позволяет выполнять операции над несколькими итерациями одновременно с использованием SIMD-инструкций.

Векторизация преобразует последовательность скалярных операций в векторные, 
каждая из которых обрабатывает несколько элементов массива одновременно. 

Эффективность цикловой векторизации определяется возможностью доказать отсутствие межитерационных зависимостей. Для этого компилятор выполняет:
\begin{itemize}
    \item анализ псевдонимов памяти (alias analysis) ~\cite{diwan1998type, gcc_alias};
    \item анализ межитерационных зависимостей (loop-carried dependence analysis) ~\cite{devan_loop_dependence};
\end{itemize}

А также вспомогательные преобразования:
\begin{itemize}
    \item раскрутку цикла (loop unrolling)~\cite{aho2006compilers, davidson2001aggressive};
    \item выделение остаточных итераций (loop peeling)~\cite{callahan1988peeling};
\end{itemize}

Реализация таких механизмов требует развитой инфраструктуры анализа зависимостей и существенно усложняет оптимизационный конвейер компилятора.

\subsection{SLP-векторизация}

SLP-векторизация ориентирована на выявление параллелизма внутри базовых блоков программы. 
В отличие от цикловой векторизации, данный подход не использует межитерационный анализ, а работает с зависимостями внутри ациклических участков кода.

Алгоритм формирует группы изоморфных независимых скалярных операций, которые могут быть преобразованы в одну векторную инструкцию.

Например:

\begin{center}
\begin{tabular}{c}
\begin{lstlisting}[language=Go, captionpos=b]
c[0] = a[0] + b[0]
c[1] = a[1] + b[1]
\end{lstlisting}
\end{tabular}
\end{center}


может быть преобразовано в одну SIMD-инструкцию сложения векторов.

SLP особенно эффективен в случаях, когда параллелизм не выражен явно через циклы.


\subsection{Сравнение подходов}

Цикловая векторизация и SLP-векторизация различаются прежде всего областью действия и требованиями к анализу зависимостей.

Цикловая векторизация охватывает всё тело цикла и нуждается в анализе межитерационных зависимостей, 
что позволяет параллельно исполнять сразу несколько итераций.
Однако это требует развитой инфраструктуры анализа зависимостей и сложных преобразований циклов.

SLP-векторизация, напротив, ограничена рамками одного базового блока:
она выявляет параллелизм непосредственно между скалярными инструкциями, группируя изоморфные операции в векторные.
Благодаря этому SLP-векторизация значительно проще в реализации и требует гораздо меньшие требования к инфраструктуре компилятора.

Для компилятора Go эти различия имеют решающее значение.
В Go отсутствует развитая инфраструктура для глубинного анализа циклов и сложных цикловых оптимизаций, 
а время компиляции является критичным требованием.
В таких условиях SLP-векторизация оказывается естественным выбором: она обеспечивает прирост производительности 
при умеренных затратах на реализацию и практически не увеличивает время компиляции.

\subsection{Метод Superword-Level Parallelism}

Метод Superword-Level Parallelism был предложен в работе Samuel Larsen и Saman Amarasinghe~\cite{Larsen2000} 
как способ выявления параллелизма на уровне независимых скалярных операций внутри базовых блоков программы. 
Основная идея метода заключается в объединении нескольких однотипных независимых скалярных операций в одну векторную.

\subsubsection{Основные этапы алгоритма}

Классический алгоритм SLP-векторизации~\cite{Larsen2000} состоит из нескольких последовательных этапов,
выполняемых в пределах одного базового блока.

\begin{enumerate}
    \item \textbf{Поиск начальных пар}. Алгоритм сканирует инструкции базового блока и ищет пары соседних обращений к памяти – смежные загрузки или сохранения данных. Такие операции могут послужить естественным началом и/или завершением параллельных последовательностей вычислений.
    
    \item \textbf{Расширение групп}. От найденных начальных пар алгоритм итеративно строит новые группы, анализируя поток данных:
    \begin{itemize}
        \item \textit{Расширение вверх}: для каждой пары инструкций в группе анализируются их аргументы – если находятся две однотипные независимые операции, они могут образовать новую пару.
        \item \textit{Расширение вниз}: рассматриваются операции, использующие результат исходных операций. Среди нескольких возможных вариантов выбирается та пара использований, которая приносит наибольшую выгоду с точки зрения модели стоимости.
    \end{itemize}
    Процесс продолжается, пока находятся новые пары, удовлетворяющие условиям независимости и изоморфности операций.
    
    \item \textbf{Объединение пар в группы}. После завершения расширения ищутся пары, которые можно объединить. Если в двух парах есть общая операция, то их можно объединить в тройки, четвёрки и т.д., формируя потенциальные векторные регистры большей ширины.
    
    \item \textbf{Планирование}. Векторные инструкции должны располагаться в корректном порядке с учётом зависимостей по данным. Выполняется топологическая сортировка групп: ни одна группа не может быть помещена в выходной поток раньше тех групп, от которых она зависит через аргументы.
    
    \item \textbf{Генерация векторного кода}. Для каждой группы формируются соответствующие векторные инструкции целевой архитектуры. 
    При необходимости группы разбиваются до поддерживаемой аппаратурой ширины. 
    Если какая-то часть вычислений используется вне векторизованного фрагмента, 
    то нужное значение скаляризуется – вставляются специальные операции извлечения отдельных данных из векторного регистра в обычный.
\end{enumerate}

Такой набор этапов гарантирует корректность векторизации и позволяет последовательно расширять функциональность.

\subsection{Особенности метода SLP}

Метод SLP обладает следующими особенностями:
\begin{itemize}
    \item ограниченная область анализа (в пределах базового блока);
    \item отсутствие необходимости анализа межитерационных зависимостей;
    \item относительно низкая сложность реализации в компиляторе;
    \item хорошая интеграция в SSA-представление.
\end{itemize}

Данные свойства делают SLP подходом, хорошо подходящим для компиляторов с ограниченной инфраструктурой оптимизаций.

Однако, метод эффективен только при наличии явно выраженных групп независимых однотипных операций. 
Если вычисления имеют сложную структуру или требуют существенного переупорядочивания данных,
эффективность SLP заметно снижается.

\subsubsection{Причины выбора SLP для компилятора Go}
Выбор метода SLP для реализации в компиляторе Go обусловлен рядом факторов:
\begin{itemize}
    \item Данный подход хорошо согласуется с существующей архитектурой компилятора.
    \item Реализация SLP требует существенно меньшего объёма инфраструктурных изменений по сравнению с цикловой векторизацией.
    \item Алгоритм обладает умеренной вычислительной сложностью, что соответствует требованию сохранения малого времени компиляции.
\end{itemize}

Таким образом, SLP представляет собой практически обоснованный компромисс между сложностью реализации и потенциальным приростом производительности.

\subsection{Выводы по главе}
В данной главе были рассмотрены аппаратные основы SIMD-вычислений, основные подходы к автоматической векторизации и детали метода Superword-Level Parallelism.
Показано, что SLP-векторизация, в отличие от цикловой, не требует анализа итерационных зависимостей и сложных трансформаций циклов. Это делает её практически применимым подходом для интеграции в компилятор Go: умеренная алгоритмическая сложность сочетается с возможностью ускорять ацикличный код, типичный для многих серверных и системных программ на Go.
